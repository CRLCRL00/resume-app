## C — Replay buffer 加密 (AES-256-GCM, opt-in)

| 特性 | 说明 |
|---|---|
| 算法 | AES-256-GCM (Node crypto, 无 dep) |
| 触发 | `SSE_REPLAY_KEY` env (64 hex chars = 32 bytes) |
| 格式 | `base64(iv 12 \|\| authTag 16 \|\| ciphertext)` |
| 性能 | ~10μs/event × 100 = 1ms, 可忽略 |
| 不设 key | plaintext passthrough (back-compat, warn once) |

**失败模式**:
- 错 key → decrypt fail → skip entry (logged)
- 改密文 → GCM auth fail → skip
- plaintext 残留 (R84-R89) → 第一次读 skip, 之后覆盖

| File | 改动 |
|---|---|
| `backend/src/db/sseReplayStore.js` | + `crypto`, `_loadEncKey`, `_encrypt`, `_decrypt`, push 走 encrypt, since 走 decrypt |
| `backend/tests/sseReplayEncrypt.test.js` | 7 tests |

**7/7 R90-C pass**:
- ✔ no key → plaintext
- ✔ key → encrypt roundtrip
- ✔ since filters by id after decrypt
- ✔ wrong key → skipped
- ✔ invalid key format → disabled
- ✔ tampered ciphertext → GCM auth fail → skipped
- ✔ mixed plaintext+encrypted → graceful
