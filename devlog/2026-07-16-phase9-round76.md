## 设计

### WeChat mp 没原生 EventSource

需要在 `wx.request` 上模拟:
- `enableChunked: true` (mp ≥ 2.10)
- `onChunkReceived` 回调拿每个 chunk
- 自己 buffer + split `\n\n` + parse each event

### Chunk 编码

mp runtime 给的 chunk 可能是 `ArrayBuffer` 或 `string`. 写了 `decodeChunk()`:
- string passthrough
- ArrayBuffer → UTF-8 decode (无 TextDecoder 依赖)

### SSE 解析

`parseEvent()` 按 spec:
- 空行 / `:` 开头 → 忽略
- `event: name` → 设置事件名
- `data: payload` → 多行累积
- `data:` 字段缺失 → **不** dispatch (per spec)
- `data:` 空字符串 → dispatch with `data=''`
- data 是 JSON → 自动 `JSON.parse`, 否则保留 string

### Dashboard 接入策略

| 阶段 | 数据源 |
|---|---|
| boot | 一次 `loadAll()` (REST) |

## 关键代码

```js
// utils/sseClient.js 核心
function sseConnect(url, opts) {
  const state = { buffer: '', closed: false };
  const task = wx.request({
    url, method: 'GET',
    header: opts.headers || {},
    enableChunked: true,
    responseType: 'text',
    success: () => { state.closed = true; opts.onClose && opts.onClose(); },
    fail: (err) => { state.closed = true; opts.onError && opts.onError(err); },
  });
  task.onChunkReceived((res) => {
    if (state.closed) return;
    state.buffer += decodeChunk(res.data);
    let idx;
    while ((idx = state.buffer.indexOf('\n\n')) !== -1) {
      const raw = state.buffer.slice(0, idx);
      state.buffer = state.buffer.slice(idx + 2);
      parseEvent(raw, opts.onEvent);
    }
  });
  return { abort: () => { state.closed = true; task.abort && task.abort(); }, get closed() { return state.closed; } };
}
```
