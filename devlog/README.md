# 开发日志 — 使用说明

## 目的

记录每日开发进度、完成事项、待办事项、踩坑笔记。每阶段结束时回看，能清晰还原整个项目演化路径。

## 文件命名

- `YYYY-MM-DD.md` — 按日期命名
- 示例：`2026-06-27.md`

## 创建方式

每天开工时：

**方式 A（手动）**

```bash
cp devlog/template.md devlog/$(date +%Y-%m-%d).md
```

**方式 B（脚本）**

```bash
bash scripts/new-day.sh
```

## 内容填写

按 `template.md` 的结构填（带量化字段）：

1. **今日目标** — 今天要完成什么（从 execution.md 当阶段任务拆）
2. **今日完成（量化）** — 表格记录：任务 / 状态（✅❌⏳） / 耗时 / commit hash
3. **进行中** — 没完成但已开工的
4. **待办** — 明天 / 后续要做
5. **阻塞问题** — 表格记录：问题 / 影响 / 责任人 / 解决方向
6. **今日进度%** — 阶段累计进度条 + 今日贡献
7. **踩坑笔记** — 遇到的坑 + 解决方法（重点！）
8. **决策记录** — 今天做了什么决定 + 原因 + 替代方案

## 节奏建议

- **开工时**：先写「今日目标」
- **每完成一项**：立刻更新「完成事项」
- **遇到坑时**：立刻写「踩坑笔记」，否则事后忘
- **下班前**：补齐所有字段，写「明日计划」

## 与文档的联动

- 阶段结束 → 更新 `docs/execution.md` 勾选完成项
- 出现新决策 → 更新 `docs/` 对应文档
- 重要踩坑 → 写入 `docs/tech.md` 或 README
