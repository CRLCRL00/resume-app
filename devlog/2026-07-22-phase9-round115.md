## 关键架构决策

### 1. 双模式 modal (wizard + assist 共存)

```xml
<block wx:if="{{wizardMode}}">
  <wizard-progress /> <wizard-question /> <input /> <wizard-next-btn />
</block>
<block wx:else>
  <R114 旧 UI: ai-chat-history + ai-suggestion-chip + modal-save />
</block>
```

`_wizardNext` 通过 `onModalSave` 模式分流:
```js
onModalSave() {
  if (this.data.wizardMode) this._wizardNext();
  else this._saveModal(...);
},
```

### 2. Wizard 状态机 + aiRequestSeq 防串话

```js
_wizardNext() {
  if (this.data.aiBusy) return;  // R114 异步竞态防护保留
  this._writeFormAndSideEffects(value);  // 不动 modal (Critical Bug 修法)
  
  const newAnswered = [...this.data.wizardAnswered, { fieldId, value }];
  const idx = FIELD_ORDER.indexOf(modalField);
  const nextFieldId = idx < FIELD_ORDER.length - 1 ? FIELD_ORDER[idx + 1] : null;
  
  if (!nextFieldId) {
    // 全部答完 → 进度 18/18, 关 modal, wizardMode=false
    this.setData({ wizardProgress: newAnswered.length, modalVisible: false, wizardMode: false });
    return;
  }
  
  this.setData({
    modalField: nextFieldId,
    modalFieldLabel: this._getFieldLabelById(nextFieldId),
    modalVisible: true,  // 关键: 保持 modal 打开
    wizardIsLoading: true,
  });
  
  this._aiRequestSeq++;  // 异步竞态防护
  // 调 AI 拿下一问...
},
```

## 踩坑笔记

### Critical Bug #1: `_saveModal` 误关 modal (T2 review)

**问题**: `_saveModal` 内含 `setData({ modalVisible: false })`, `_wizardNext` 复用 → 用户答第 1 问点"下一问" → modal 关闭 → wizard 只问 1 问就退出. chips/picker 字段同样命中 bug.

**解决**: 抽 `_writeFormAndSideEffects(value)` 纯写 form + 副作用, 不动 modal. 让 `_wizardNext` 调它, 然后 setData 显式 `modalVisible: true` 切下一字段.

**教训**: 复用函数前先看副作用 — 通用函数会无意中改 caller 不想要的 state. 拆分为"纯函数 + 关 modal 函数"更安全.

### Important #2: 字段总数硬编码错误

**问题**: `wizardTotal: 14` 硬编码, 实际字段数 = 18 (basic 4 + education 4 + work 5 + expected 4 + skills 1). 进度条 "5 / 14 = 35.7%" 视觉撒谎, 实际是 5/18 = 27.8%.

**解决**: 在 CONSTELLATIONS 后派 `FIELD_ORDER` + `FIELD_COUNT` 常量. data 用 `wizardTotal: FIELD_COUNT`.

**教训**: 派生常量 vs 硬编码 — 字段定义在 CONSTELLATIONS, 总数必须从同一处派生.

### Important #3: wizard 模式仍调 LLM (浪费)

**问题**: `onModalInput` 不管 wizardMode 都 debounce `_aiSuggest`, 后端返 R114 形状 (opening/followups/suggestion), wizard WXML 不显示 → 浪费 token + LLM 调用 + `aiBusy` 锁死 `_wizardNext` 静默丢弃.

**解决**: `onModalInput` 开头加守卫:
```js
if (this.data.wizardMode) {
  this.setData({ modalValue: e.detail.value });
  return;
}
```

**教训**: 一个 handler 多种模式时, 必须按 mode 分流. 否则会同时跑两套逻辑互相干扰.

### Important #4: 测试盲区

**问题**: 2 条 R115 测试只查字符串存在性 (`assert wxml includes 'wizard-progress'`), 抓不到任何 Critical Bug.

**解决**: 加 3 条防回归测试, 解析代码结构:
- `_writeFormAndSideEffects` 必须存在 + `_wizardNext` 必须调它 + 切下一字段时必须 `modalVisible: true`
- `wizardTotal` 必须派生自 `FIELD_COUNT` + 不能硬编码 14
- `onModalInput` 必须有 `if (this.data.wizardMode)` 守卫

**教训**: 字符串存在性测试 = 0 抓 regression. 行为 / 结构断言才有效.
