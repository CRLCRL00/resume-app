## R78 — Auto-reconnect + exponential backoff

### 之前 (R76)
- `onError` 里 `setTimeout 10s` 后手动重连 — 简单但不优雅

### 之后 (R78)
- 新 `sseConnectWithRetry(url, opts)` 包装 `sseConnect`
- 配置: `backoffMs=1000`, `maxBackoffMs=30000`, `maxAttempts=0` (无限)
- Backoff 序列: 1s → 2s → 4s → 8s → 16s → 30s (cap) → 30s ...
- `shouldReconnect()` predicate — 切到 compact 模式自动停
- `onStatus` 5 状态: `connecting` / `open` / `waiting` / `exhausted` / `stopped`
- `stop()` 立即终止 (清 timer + abort task)

### mp dashboard 接入
```js
this._sseAbort = sseConnectWithRetry(`${apiBaseUrl}/api/admin/dashboard/stream`, {
  headers: ...,
  backoffMs: 1000, maxBackoffMs: 30000, maxAttempts: 0,
  shouldReconnect: () => this.data.mode === 'fullscreen',
  onStatus: (status) => this.setData({ sseStatus: status }),
  onEvent: (e) => this._onSseEvent(e),
});
```

### 数据流 (R78)
```
SSE connect fail
  ↓ attempt=1, status='waiting', delay=1000ms
1s 后 reconnect → fail → attempt=2, delay=2000ms
2s 后 → fail → 4s → 8s → 16s → 30s → 30s ... (cap)
  ↓
shouldReconnect() === false (compact mode) → status='stopped'
或 stop() → status='stopped'
```
