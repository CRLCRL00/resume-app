### 2. R106b 教训 — 预计算 fieldState, 不用 inline function

R116 T1 重新引入 inline function (`_isFieldFilled(field.id)`), R106b 实测证明会断整个 view 渲染.

修法:
```js
// js _buildFieldStates helper
_buildFieldStates() {
  return CONSTELLATIONS.map((c) => ({
    id: c.id,
    name: c.name,
    fields: c.fields.map((f) => ({
      ...f,
      fieldState: {
        filled: this._isFieldFilled(f.id),
        current: this.data.currentFieldId === f.id,
        done: FIELD_ORDER.indexOf(f.id) < this.data.currentFieldIndex,
        value: this._getFieldValue(f.id) || '',
      },
    })),
  }));
},
```
```xml
<!-- wxml — 无 inline function -->
<view class="feed-field-card {{field.fieldState.filled ? 'filled' : ''}} ...">
```

## 踩坑笔记

### Critical Bug 1: WXML inline function (R106b 教训)

**问题**: R116 T1 wxml line 26 `class="... {{_isFieldFilled(field.id) ? 'filled' : ''}} ..."` — R106b 真机截图证明 inline function 会断整个 view 渲染.

**解决**: 预计算 fieldState 到 data, wxml 用字段访问 `{{field.fieldState.filled}}`.

**教训**: 重新引入 R106b 已修过的 bug. T1 implementer 没看 R106b fix 注释. 防回归测试加 `R116 fix: wxml 无 {{_isFieldFilled}} / {{_getFieldValue}} / {{_getFieldIndex}}` 3 条.

### Critical Bug 2: R99 modal-ai-bubble 未授权删

**问题**: R116 T1 implementer 重写 modal 部分时, 把 R99 modal-ai-bubble 元素删了 (不只是删 R98, 是改了 R99 设计).

**解决**: 恢复 R99 modal-ai-bubble + 加 `aiHistory.length === 0` 条件避免与 ai-chat-history 重复.

**教训**: implementer 删任何元素前需要明确授权, 不能"顺手"删已 commit 的功能.

### R103 测试翻转补充

**问题**: R103 → R116 翻转测试断言 "不应再调 createSelectorQuery". R116 T2 _snapToSection 又用了 createSelectorQuery (用于 snap 而非 _drawLines).

**解决**: 翻转测试改为检查 _drawLines 函数体内不调 createSelectorQuery (限定 _drawLines, 允许其他用途).
