# 开发日志 — 2026-07-16（Phase 9 Round 85 — Redis INCR event id）

> 阶段：Phase 9 R85 — 多 pod 安全 event id (atomic, shared)
> 前置：[2026-07-16-phase9-round84.md](../devlog/2026-07-16-phase9-round84.md)

## 起点

R84 Redis buffer OK, 但 `_eventId` 仍是 process-local. 多 pod 时 id 冲突. user 答"A" → R85.

## 问题

```
pod A: _eventId = 1, 2, 3 (process-local)
pod B: _eventId = 1, 2, 3 (separate process)
                      ↑ 冲突!

client 重连 Last-Event-ID: 2:
  - pod A: replay events with id > 2 (own pod's events)
  - pod B: replay events with id > 2 (own pod's events)
  - 但 same id 在不同 pod 意义不同 → 漏/重
```

## 解法 — Redis INCR

```js
// sseReplayStore.js
async function nextEventId() {
  try {
    const id = await getRedis().incr(EVENT_ID_KEY); // sse:event:id
    return Number(id);
  } catch (e) {
    _localEventId += 1;  // fallback (single-pod only)
    return _localEventId;
  }
}
```

### 特性

| | before (R82) | after (R85) |
|---|---|---|
| counter 位置 | process memory | Redis |
| 多 pod 共享 | ❌ 重叠 | ✅ atomic |
| restart 连续 | ❌ 归 0 | ✅ INCR 继续 |
| Redis 挂 | n/a | fallback local (degraded) |
| 性能 | 0ns | ~1ms RTT |

## 关键代码

```js
// sseReplayStore.js
const EVENT_ID_KEY = 'sse:event:id';

async function nextEventId() {
  try {
    const id = await getRedis().incr(EVENT_ID_KEY);
    return Number(id);
  } catch (e) {
    _localEventId += 1;
    logger.warn({ err: e.message }, 'sse event id: Redis INCR failed, using process-local fallback');
    return _localEventId;
  }
}

async function currentEventId() {
  try {
    const v = await getRedis().get(EVENT_ID_KEY);
    return v ? Number(v) : null;
  } catch (_) {
    return _localEventId || null;
  }
}

// dashboardStream.js — 3 call sites
const eid = await nextEventId();  // was: const eid = nextEventId();
```

## Multi-pod scenarios

| scenario | behavior |
|---|---|
| 1 pod (R40+ 现状) | INCR +1 each event, 单调递增 |
| 2+ pods, Redis up | INCR atomic, both pods 拿不同 id (interleaved) |
| 2+ pods, Redis 挂 | fallback local, 各自从 1 开始 (still unique within pod) |
| pod restart | INCR 继续 (Redis 保留 counter) |
| Redis 重启 (data lost) | INCR 重新从 1 (potential collision with old events) — 接受 |
