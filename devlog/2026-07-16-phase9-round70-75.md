## R72 — SSE dashboard push

### Design
- `backend/src/routes/admin/dashboardStream.js` (新文件)
- `GET /api/admin/dashboard/stream` → `text/event-stream`
- 10s 推 `{overview, ts}`, 15s heartbeat `{ts}`
- 自动 res.close 清理 (req.on('close'))
- mp 客户端: 用 `wx.request` + onChunkReceived 解析 (无需 WebSocket 库)

### 优势 vs 30s polling
- 实时 (10s vs 30s)
- server-side push (无需 client 重连)
- 单连接 (vs N client 各 N request)
- auto-reconnect built-in

### 没做
- **mp 客户端 SSE 接入** — R72 只 backend, 前端接入留 R76+

## R73 — Match explanation

### Endpoint
`GET /api/match/:matchId/explain` → 返回 factor breakdown

### Factor 模型 (sums to 100)
| factor | weight | 评分 |
|---|---|---|
| city | 25 | exact match = 25, mismatch = 0 |
| salary | 25 | ratio ≥ 1.0 = 25, 0.8-1.0 = 15, <0.8 = 5 |
| degree | 20 | 满足 = 20, 不满足 = 0 |
| llm_reason | 30 | 有 reason = 30, 无 = 0 |

未提供期望值 → 给 half score (12/12/10/15 = 49 baseline).

### Use case
用户点匹配列表中某条 → 弹 modal 显示 factor 列表 + 各自评分 → 知道为什么.

## R74 — User segments

### Endpoint
`GET /api/admin/dashboard/user-segments`

### Segments
| key | 条件 | color |
|---|---|---|
| active | last_login ≤ 7d | 绿 |
| recent | 8-30d | 蓝 |
| dormant | 31-90d | 黄 |
| inactive | >90d | 红 |
| never | NULL | 灰 |
| new | created ≤7d (与 active 重叠独立报) | 紫 |

### Use case
- 运营周报: "本周流失 12 人 (从 active → inactive)"
- 推送目标: "召回沉睡用户 (>30d)" → 调 wx mp 模板消息
