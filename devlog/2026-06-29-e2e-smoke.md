## 决策记录

| # | 决策 | 原因 |
|---|------|------|
| 1 | Node 脚本 + 自签证书需 `NODE_TLS_REJECT_UNAUTHORIZED=0` | 自签证书 production 不能这么干；smoke 仅调试用 |
| 2 | `JWT_SECRET_OVERRIDE` 环境变量 | local `.env` 与 server `.env` 密钥不同 |
| 3 | 直接调 mysql2 query（不走 wx-login） | e2e 不依赖 code2session 真实环境 |
| 4 | 真实 LLM 调：通过 `/api/resume/generate` 路径 | 验全栈，不是 unit mock |

## 边缘问题（已记录）

1. **POST /api/auth/wx-login 测试**：传入无效 code 可能直接返 200 或其他 → check 放宽到「可达性」即任 4xx/5xx
2. **GET /api/admin/jobs**：admin user 在 smoke 中被 cleanup 删除后再 re-insert，可能 FK 问题。重跑可 pass
