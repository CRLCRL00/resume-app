## 关键架构决策

### 1. 统一存储 helper (_saveAuth)

```js
app.js:
_saveAuth(data) {
  if (!data) return;
  if (data.token) wx.setStorageSync('token', data.token);
  if (data.refreshToken) wx.setStorageSync('refreshToken', data.refreshToken);
  if (data.user) wx.setStorageSync('user', data.user);
},

// login / devQuickLogin 改用:
this._saveAuth(res.data.data);
```

效果: access token + refresh token + user 三个一起存, 未来加 token type 也只改一处.

### 2. 401 自动 refresh (utils/request.js)

```js
if (res.statusCode === 401 && !opts._retried401) {
  const app = getApp ? getApp() : null;
  if (app && typeof app.refreshAccessToken === 'function') {
    app.refreshAccessToken().then((r) => {
      if (r && r.ok) {
        // 用新 token 重试原请求
        doRequest(Object.assign({}, opts, { _retried401: true })).then(resolve, reject);
      } else {
        clearToken();
        reject(res.data);
      }
    });
  }
}
```

效果: 用户表单中途 token 过期, 前端自动 refresh, 用户无感.

## 踩坑笔记

### Nit #1: regex 非贪婪匹配过早结束

**问题**: 测试用 `login\s*\(\s*\)\s*\{[\s\S]*?wx\.login`, 但非贪婪 `*?` 只 match 到第一个 `wx.login` (line 95), 不含后面的 `_saveAuth` (line 103).

**解决**: 改 regex 终点为 `this._saveAuth`:
```js
/login\s*\(\s*\)\s*\{[\s\S]*?this\._saveAuth/
```

**教训**: regex 测试跨函数时, 终点要选函数内关键 token, 不能选早出现的字符串.
