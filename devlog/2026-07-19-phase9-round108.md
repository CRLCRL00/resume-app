## 关键架构: 嵌套 wrapper 解决 transform 冲突

R107 T4 给 `.constellation` 加 `spin-slow` (rotate), R108 T1 计划给同一 selector 加 `constellation-breathe` (scale). **冲突**: transform 属性后写覆盖前写, 两个动画互相打断.

**解决**: WXML 多嵌一层 `.constellation-rotate` 内层, 让 scale 走外层 / rotate 走内层, 互不干扰:

```xml
<!-- before -->
<view class="constellation constellation--{{con.id}}">
  <view class="const-label">...</view>
  ...
</view>

<!-- after -->
<view class="constellation constellation--{{con.id}}">     <!-- scale 呼吸 -->
  <view class="constellation-rotate">                       <!-- R107 T4 spin -->
    <view class="const-label">...</view>
    ...
  </view>
</view>
```

WXSS 也跟着迁移选择器: `.starfield.ready .constellation` → `.starfield.ready .constellation-rotate`.

## 2 道闸 复盘 — 一次正面教学

| Task | spec-review | code-quality-review | 备注 |
|---|---|---|---|
| T1 星座呼吸 | PASS 一发 | PASS 一发 | — |
| T2 粒子拖尾 v1 | PASS | **FAIL** (animation-play-state: paused 不能释放 transform, scale 仍被 R107 T4 锁住, 拖尾不动) → implementer amend 改成 `.starfield.touching .particle { animation: none }` (animation: none 才是 CSS 规范层面的真正 reset) | **关键教学点** — v1 fix 通过了 spec-review 却被 code-quality 抓到真 bug |
| T2 粒子拖尾 v2 | — | PASS (animation: none 真修 + 简化 catchtouchmove 单绑避免 bind+catch 冲突) | 最终 commit `68b4c60` |

**v1→v2 fix 是一次正面教学案例**:
- spec-review 闸只审"是否符合 plan", 而 plan 里我**写错了** CSS 规范 (以为 `animation-play-state: paused` 能释放 transform, 实际 paused 只停动画不重置 transform).
- code-quality-review 闸从浏览器/CSS 规范层面拦下来: paused 不等于 none, 不能用.
- 如果没有 code-quality 这道闸, v1 fix 已经 push 上 IDE, 用户真机测会发现"粒子不跟手指拖"——spec 看起来对, 行为不对.
- 教训: CSS 状态变更/规范类问题, spec-review 看不到, 必须靠 code-quality-review.

另一个 v2 细节: 原本用 `bindtouchmove + catchtouchmove=""` 两套, 后改成只用 `catchtouchmove` 单绑 — bind+catch 共存时优先级混乱 (catch 已 bind 的不会冒泡, 但 bind 之后的 catch 会清 tap).
