## 踩坑笔记（新增）

### 本地 Redis 没装为服务

**问题：** `redis-server --service-start` 报 error 1060（服务不存在）。

**原因：** Redis on Windows 是便携版，从未 `redis-server --service-install` 过。

**解决：** 临时用 `redis-server.exe redis.windows.conf --daemonize yes` 后台跑。要开机自启得用 NSSM 或 task scheduler 包装。

**教训：** Windows 服务的便携性差，开发期别太依赖 service 模式。

### config 校验太严导致本地起不来

**问题：** `REDIS_PASSWORD=`（空字符串）触发 `Missing required env: REDIS_PASSWORD`，所有测试都红。

**原因：** 原来把 `REDIS_PASSWORD` 写进 REQUIRED，但 Redis 本地无密码、服务器有密码，约束不该一样强。

**解决：** 从 REQUIRED 移除，空串统一转 `undefined`，ioredis 收到 undefined 时不发 AUTH。

**教训：** 「必填」分两类：业务上必有（如 DB_PASSWORD、API_KEY），连接器上可有可无（如 Redis 密码，开发机常裸跑）。后者从 REQUIRED 拿掉，转为「空 = 不传」语义。
