```js
// backend/src/routes/ai.js (wizard 分支)
任务:
```

### 3. 预计算 currentRecommendation (R106b 教训)

wxml 用 `{{currentRecommendation.value}}` 而非 `{{recommendations[currentRecommendationIdx].value}}`:
- 避免 wxml inline array index (R106b 教训说 inline 表达式会断 view)
- _buildTinderState helper 在 setData 时同步算

```js
_buildTinderState(overrides = {}) {
  const recs = overrides.recommendations !== undefined ? overrides.recommendations : (this.data.recommendations || []);
  const idx = overrides.currentRecommendationIdx !== undefined ? overrides.currentRecommendationIdx : (this.data.currentRecommendationIdx || 0);
  return {
    recommendations: recs,
    currentRecommendationIdx: idx,
    currentRecommendation: recs[idx] || null,
    ...overrides,
  };
},
```

## 踩坑笔记

### Nit #1: stubChatJson 格式错 (R115 旧坑再踩)

**问题**: R117 测试用 `stubChatJson({parsed: ...})` sync 对象, 但 helper 期望 `stubChatJson(async () => ({parsed: ...}))` async 函数.

**解决**: 改用 `stubChatJson(async () => ({parsed, usage}))` — R115 T1 implementer 已经踩过.

### Nit #2: regex 早终止 (R115 旧坑再踩)

**问题**: 测试 regex `login\s*\(\s*\)\s*\{[\s\S]*?wx\.login` 非贪婪 match 只到第一个 `wx.login` (不含 `_saveAuth`).

**解决**: 改 regex 终点为 `this._saveAuth`: `login\s*\(\s*\)\s*\{[\s\S]*?this\._saveAuth`.
