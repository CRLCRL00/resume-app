## 起点

R80 metrics 暴露. user 答"A" → R81 cap.

## 问题

即使有 coalesce (R79), 一个 admin 多 tab:
- 5 tab × (snapshot/10s) = 5 setData/10s in mp client
- mp runtime 网络栈压力
- 若 admin openid 暴露 → 攻击者可开 N conn

## 设计

### Per-admin cap = 5

```js
const MAX_CONNECTIONS_PER_ADMIN = 5;

if (_connCount(conn.openid) >= MAX_CONNECTIONS_PER_ADMIN) {
  sseRejectedConnections.inc({ reason: 'per_admin_cap' });
  return res.status(429).json({
    code: 1429,
    message: 'too many concurrent SSE connections for this admin (max 5)',
    current,
    cap: MAX_CONNECTIONS_PER_ADMIN,
  });
}
```

### Registry refactor: Set → Map<openid, Set>

之前: `connections: Set<conn>`
之后: `connectionsByAdmin: Map<openid, Set<conn>>`

优点:
- O(1) 查某 admin 当前 conn 数
- 自动清理空 Set (admin 全断开时)
- Ticker 遍历仍 O(N) 但分组清晰

### 新 metric

| metric | type | labels | 用途 |
|---|---|---|---|
| `sse_rejected_connections_total` | Counter | `reason` | 拒接计数 (per_admin_cap / future reasons) |

### `_stats()` 升级

```js
return {
  connections: _totalCount(),
  admins: connectionsByAdmin.size,
  perAdmin: { 'admin_openid_a': 2, 'admin_openid_b': 5 },
  lastSnapshotAt,
  cacheAgeMs,
};
```
