## 根因

### Bug A: WXML 外层 `wx:for-item="con"` 但内部引用 `item.*`

```xml
<view wx:for="{{constellations}}" wx:key="id" wx:for-item="con">
  <view class="const-halo"
        style="left: {{item.cx - 50}}px; top: {{item.cy - 50}}px; ...">
```

微信小程序对 `wx:for-item` 重命名后的 `item.*` 引用有 fallback 但...
在 libVersion 3.16.1 + 真机/IDE 3.16.1 引擎上**叠加 inline 函数调用**会让整段 view 子树完全不渲染.

### Bug B: WXML inline 函数调用 `_isFieldFilled(item.id) ? 'filled' : ''`

微信 WXML inline 函数调用本来就脆弱 — 实测叠加 Bug A 后触发整个星座 view 不渲染.

## 修复

### 改动文件

| 文件 | 改动 |
|---|---|
| `pages/form/bigscreen/bigscreen.wxml` | 外层 `{{item.*}}` → `{{con.*}}`; 删 inline `_isFieldFilled()`; class 用 `{{particle.filled ? 'filled' : ''}}` |
| `pages/form/bigscreen/bigscreen.js` | `layoutParticles` 输出每个粒子带 `filled` 字段 (默认 false); 加 `_refreshParticleFilled()` helper; `_saveModal` setData 回调里调它 |
| `tests/bigscreen.test.js` | +3 测试: ① WXML 不再有 `{{item.*}}` 引用 ② WXML 不再有 inline 函数 + 改用 `particle.filled` ③ js `layoutParticles` 输出含 `filled` 字段 |

### 关键代码 (js)

```js
// layoutParticles: 提前把 filled 加进数据
const particles = c.fields.map((f, j) => ({
  ...f,
  x: ccx + Math.cos(partAngle) * partR,
  y: ccy + Math.sin(partAngle) * partR,
  filled: false,  // R106b: 首次 layout false; _initLayout 后由 _refreshParticleFilled 重算
}));

// 新 helper: 用 form 数据重算每个粒子 filled 视觉态
_refreshParticleFilled() {
  const constellations = (this.data.constellations || []).map((c) => ({
    ...c,
    particles: (c.particles || []).map((p) => ({
      ...p,
      filled: this._isFieldFilled(p.id),
    })),
  }));
  this.setData({ constellations });
}

// _saveModal: setData 完成回调里调一次
this.setData({ form, skillsCount, completion, modalVisible: false, ... },
  () => {
    this._refreshParticleFilled();  // R106b
    this._drawLines(this.data.width, this.data.height);
  });
```
