## 设计

### Server emits `id:` on every event

```
id: 42
event: dashboard-update
data: {"ts":..., "overview":{...}}

```

### Client sends `Last-Event-ID` on (re)connect

```http
GET /api/admin/dashboard/stream HTTP/1.1
Last-Event-ID: 42
```

Server reads header, logs `sse: client resumed with Last-Event-ID`.

### Why now (vs R72+)

- mp sseClient 之前无 `Last-Event-ID` 概念
- 添加成本: ~30 行 client + ~10 行 server
- 价值: 断线重连时不丢审计 trail (server 知道 client 从哪续)

### MVP scope

- ✅ Server 发 `id:` 字段
- ✅ Server 读 `Last-Event-ID` header (log)
- ✅ Client 缓存 lastEventId
- ✅ Reconnect 时自动发
- ⏳ **Replay** 实际事件历史 — 留 R83+ (dashboard 快照幂等, replay 非阻塞)

## 关键代码

```js
// server
let _eventId = 0;
function nextEventId() { return ++_eventId; }
// ...
res.write(`id: ${nextEventId()}\nevent: dashboard-update\ndata: ${...}\n\n`);

// on connect
const conn = {
  res,
  openid: (req.user && req.user.openid) || 'unknown',
  lastEventId: req.headers['last-event-id'] || null, // R82
};
if (conn.lastEventId) {
  logger.info({ openid, lastEventId: conn.lastEventId }, 'sse: client resumed');
}
```
