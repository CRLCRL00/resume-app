## 设计

### Redis LIST (LPUSH + LTRIM)

```
key: sse:replay:buffer
order: LPUSH adds to head (newest at index 0)
cap: LTRIM 0 (REPLAY_BUFFER_SIZE - 1) keeps only most recent 100
read: LRANGE 0 -1 returns oldest→newest (need reverse for chronological)
```

### Memory math
- 100 events × ~1KB = **100KB Redis memory** (negligible)
- TTL: none (always kept; cleared only by clear())
- Survives: pm2 restart, deploys, crash recovery

### Failure modes

| scenario | behavior |
|---|---|
| Redis push 失败 | in-memory fallback buffer (last-ditch); 推下个事件时再试 |
| Redis read 失败 (since) | empty replay; client 收 current snapshot |
| Redis 临时挂 | in-memory 暂存, Redis 恢复后 in-memory 数据不再回填 (lost) |

### Data flow

```
server on tick:
  const eid = nextEventId();
  const snap = await getSharedSnapshot();
  res.write(`id: ${eid}\nevent: ...\n\n`);
  bufferEvent(eid, ...) // → replayStore.push → Redis LPUSH+LTRIM (1 round-trip)
  ↓
client reconnect with Last-Event-ID: 5:
  server: replayStore.since(5) → Redis LRANGE → filter id>5 → reverse → replay
  ↓
client: onEvent for each replay → state update → 接上 live
```

## 关键代码

```js
// db/sseReplayStore.js
async function push({ id, event, data, ts }) {
  const payload = JSON.stringify({ id, event, data, ts });
  try {
    await getRedis().multi()
      .lpush(REPLAY_BUFFER_KEY, payload)
      .ltrim(REPLAY_BUFFER_KEY, 0, REPLAY_BUFFER_SIZE - 1)
      .exec();
    return true;
  } catch (e) {
    fallbackBuffer.unshift({ id, event, data, ts });
    if (fallbackBuffer.length > FALLBACK_CAP) fallbackBuffer.length = FALLBACK_CAP;
    return false;
  }
}

async function since(sinceId) {
  const target = Number(sinceId);
  const raw = await getRedis().lrange(REPLAY_BUFFER_KEY, 0, REPLAY_BUFFER_SIZE - 1);
  const items = [];
  for (let i = raw.length - 1; i >= 0; i--) {  // reverse for chronological
    const obj = JSON.parse(raw[i]);
    if (Number(obj.id) > target) items.push(obj);
  }
  return items;
}
```

## 已知限制

- **`_eventId` 仍是 process-local**: 多 pod 部署时, 每个 pod 独立计数 (可能 id 重叠, replay 误判)
  - 现状: 单 pod pm2 (R40+), 不影响
  - 多 pod 后: 需 Redis INCR 替代 `_eventId` 计数 (R85+ 留)
- **Fallback in-memory** 仅短暂 (进程内), Redis 恢复后不自动回填
