## 1. 信息架构

### 用户端页面树

```
pages/
├── index         首页（动态按钮 + 静默登录）
├── form          资料表单（5 步 + 草稿持久化）
├── resume        简历预览（Markdown 渲染）
├── jobs          岗位列表（匹配结果）
└── job           岗位详情

admin/  ← 分包
└── pages/
    ├── job-list     岗位管理列表
    ├── job-edit     岗位编辑
    └── prompt-edit  Prompt 编辑
```

## 3. 关键页面规范

### 3.1 首页

- 顶部：欢迎语（昵称有则展示，无则「你好」）
- 主区域：2 个主按钮，居中堆叠
  - 无简历：「填写资料生成简历」「查看岗位推荐」
  - 有简历：「更新我的简历」「查看匹配岗位」
- 底部：状态提示（已生成 N 份简历 / 最近匹配时间）

### 3.2 资料表单（5 步）

- Step 1：最高学历（学校 / 专业 / 学位 / 起止）
- Step 2：工作经历（可多条：公司 / 职位 / 起止 / 描述）
- Step 3：技能点（标签预置 + 自定义 + 去重 + 长度 ≤20 字）
- Step 4：期望（城市 / 薪资 / 岗位）
- Step 5：预览提交

**降级方案：** MVP 阶段先做 3 步核心（教育 + 经历 + 期望信息），技能用文本输入，跑通后回头优化标签体验。

**草稿：** 每步切换自动 `wx.setStorageSync('resume_form_draft')`，进入时优先回填。

### 3.3 简历预览

- 顶部 Markdown 渲染区（towxml）
- 底部固定操作栏：
  - 「换一版表述」（30s 防抖 + 日 3 次限流）
  - 「修改资料」（回表单）
  - 「找岗位」（直接调匹配）

### 3.4 岗位列表

- 卡片信息：公司名 + 岗位名 + 城市 + 薪资 + 匹配分圆环 + 文字 + 一句话理由
- 加载状态：
  - 「正在筛选合适岗位...」（<1s，粗筛阶段）
  - 「AI 正在分析匹配度，请稍候...」（LLM 阶段）
  - 「匹配数据较多，预计还需几秒」（>15s 追加）
- 空状态：「当前条件下无匹配岗位」+ 「去修改期望」按钮
- 下拉刷新：清当前列表缓存重跑

### 3.5 岗位详情

- 顶部：匹配分大字 + 个性化理由
- 中部：岗位描述（Markdown）
- 底部：公司 + 「复制岗位信息」按钮（一键复制 公司+岗位+城市+薪资）

### 3.6 管理端

- 入口：「我的」页面底部「管理后台」按钮
- 前端拦截：`onLoad` 调 `GET /api/admin/check`，未通过跳转首页 + toast
- 后端拦截：所有 `/api/admin/*` 走管理员鉴权中间件
- 列表页：表格 + 上下架开关 + 编辑 / 软删 / 恢复
- 编辑页：全字段表单 + Markdown 预览（管理端）
- Prompt 页：两个 textarea + 「保存」+ 「保存并清空所有匹配缓存」

## 4. 交互规范

| 场景 | 反馈 |
|------|------|
| 表单提交中 | loading mask + 禁用按钮 |
| LLM 调用中 | 分阶段文案 + 35s 前端超时 |
| 接口 401 | 清除本地 token + 跳转首页 |
| 接口 429 | toast 区分「今日次数用完」/「操作太频繁」 |
| 接口 5xx | toast「服务异常，请稍后重试」+ 失败重试按钮 |
| 复制成功 | toast「已复制」 |
| 空状态 | 文字说明 + 行动按钮 |

## 5. 数据规范

- 薪资字段统一「K/月」，前端展示加单位
- 时间字段统一 `YYYY-MM` 或 `YYYY-MM-DD`
- 学历层级：大专 < 本科 < 硕士 < 博士
- 工作年限映射：应届=0 / 1-3 年=2 / 3-5 年=4 / 5+ 年=6 / 不限=null

## 5.1 粗筛维度字段表

匹配流程的 SQL 粗筛阶段使用下表的硬过滤 + 软排序规则。

### 硬过滤（AND 关系，必须全部满足才进候选集）

| 维度 | 字段 | 类型 | 规则 |
|------|------|------|------|
| 城市 | `jobs.city` | 字符串 | 精确匹配用户 `resumes.expected_city`；若用户填「不限」则跳过 |
| 薪资下限 | `jobs.salary_min` | 整数 | ≤ 用户 `expected_salary_max`（岗位最低薪资不超过用户期望上限） |
| 薪资上限 | `jobs.salary_max` | 整数 | ≥ 用户 `expected_salary_min`（岗位最高薪资不低于用户期望下限） |
| 学历 | `jobs.degree_required` | 字符串 | 学历层级兼容匹配（见下表） |
| 经验 | `jobs.experience_required` | 字符串 | 工作年限区间匹配（见下表） |
| 上架状态 | `jobs.is_online` | 0/1 | = 1 |
| 未删除 | `jobs.is_deleted` | 0/1 | = 0 |

### 软排序（精排前的优先级，desc 顺序叠加）

| 排序键 | 权重 | 计算方式 |
|--------|------|----------|
| 技能命中数 | 60% | `JSON_CONTAINS(jobs.skills_required, ...)` 命中数量 |
| 岗位名关键词命中 | 25% | `jobs.title LIKE '%{expected_position}%'` 命中 1 加分 |
| `sort_weight` | 15% | 运营干预权重（直接加到分数） |

### 学历层级兼容表

| 岗位要求 | 用户最低学历可接受 |
|----------|---------------------|
| 不限 | 任意 |
| 大专 | 大专 / 本科 / 硕士 / 博士 |
| 本科 | 本科 / 硕士 / 博士 |
| 硕士 | 硕士 / 博士 |
| 博士 | 博士 |

### 工作年限兼容表

| 岗位要求 | 用户工作年限范围 |
|----------|------------------|
| 不限 | 任意 |
| 应届 | ≤ 1 年 |
| 1-3 年 | 1 ~ 3 年 |
| 3-5 年 | 3 ~ 5 年 |
| 5+ 年 | ≥ 5 年 |

### 粗筛输出

- 排序后取 Top 15
- 不足 15 时取全部（兜底）
- 0 结果时返空 + 友好提示（不进 LLM 阶段）

> **配置位置**：粗筛 SQL 实现在 `backend/src/services/match.js` 的 `coarseFilter()` 函数；阈值常量集中在文件顶部便于调优。

## 6. 接口设计规范

- RESTful 风格
- 路径前缀 `/api/`
- 用户端 `/api/`，管理端 `/api/admin/`
- 成功：`{ code: 0, data: ... }`
- 业务错误：`{ code: 1xxx, message: "..." }`，HTTP 状态码语义保留
- 限流：`HTTP 429` + `{ code: 1001, message: "今日次数已用完" }`
- 未授权：`HTTP 401` + `{ code: 1002, message: "请重新登录" }`
- 无权限：`HTTP 403` + `{ code: 1003, message: "无权限" }`
