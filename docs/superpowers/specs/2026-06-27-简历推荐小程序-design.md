## §1 总体架构

```
┌─────────────────────────────────────────────────────────┐
│  微信小程序 (用户端)                                      │
│  - 资料填写表单（教育/经历/技能/期望城市/薪资）              │
│  - 简历预览页（LLM 生成结果）                              │
│  - 岗位推荐列表（公司+岗位+匹配分+理由）                    │
└──────────────────┬──────────────────────────────────────┘
                   │ HTTPS (wx.request)
                   ▼
┌─────────────────────────────────────────────────────────┐
│  腾讯云轻量服务器 (后端)                                  │
│  Node.js + Express + MySQL + Redis                       │
│  - POST /api/auth/login         → code2session          │
│  - POST /api/resume/generate    → LLM 生成简历           │
│  - POST /api/match              → 两阶段匹配             │
│  - GET  /api/jobs/:id           → 岗位详情               │
│  - /api/admin/*                 → 管理端（鉴权隔离）      │
│  - 缓存层：Redis (列表+单岗位，24h)                      │
└──────────────────┬──────────────────────────────────────┘
                   │
       ┌───────────┴───────────┐
       ▼                       ▼
┌─────────────┐         ┌─────────────┐
│  MySQL      │         │  DeepSeek   │
│  - users    │         │  LLM API    │
│  - resumes  │         │  (OpenAI    │
│  - jobs     │         │  兼容)      │
│  - matches  │         └─────────────┘
│  - admins   │
│  - prompts  │
│  - admin_operation_logs │
│  - 列表+单岗位缓存     │
└─────────────┘

┌─────────────────────────────────────────────────────────┐
│  微信小程序 (管理端分包)                                  │
│  - 岗位 CRUD（增删改查 / 上下架 / 软删）                  │
│  - Prompt 读 / 写                                         │
│  - 双层鉴权：onLoad 拦截 + 后端中间件                    │
└─────────────────────────────────────────────────────────┘
```

### 关键决策

- **LLM 选 DeepSeek**：中文强、¥0.001/千 token 量级、OpenAI 兼容协议
- **缓存分两层**：列表级 `match:list:{uid}:{rid}` + 单岗位 `match:{uid}:{jid}`
- **MySQL matches 表只增不删**：持久化历史，Redis 才是缓存
- **登录用 code2session 后端换**：前端不传 openid
- **同机部署**：2C4G 跑 Node + MySQL + Redis，资源硬约束防 OOM

### MVP 不做

- 用户登录 / 手机号
- PDF 导出 / 投递
- 推送通知 / 多语言

## §2 数据模型

### users

```sql
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  openid VARCHAR(64) UNIQUE NOT NULL,
  nickname VARCHAR(64),
  avatar_url VARCHAR(512),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX (openid)
);
```

### resumes

```sql
CREATE TABLE resumes (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT UNSIGNED NOT NULL,
  education JSON,              -- [{school, major, degree, start, end}]
  experience JSON,             -- [{company, title, start, end, description}]
  skills JSON,                 -- ["React", ...]
  expected_city VARCHAR(64),
  expected_position VARCHAR(128), -- 期望岗位（粗筛维度）
  expected_salary_min INT,     -- 单位 K/月
  expected_salary_max INT,
  raw_input TEXT,
  llm_output MEDIUMTEXT,       -- Markdown
  version INT NOT NULL DEFAULT 1,
  is_active TINYINT UNSIGNED NOT NULL DEFAULT 1, -- 业务层保证唯一活跃
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX (user_id, is_active)
);
```

> **业务层不变量**：用户新增简历版本时，先 UPDATE 该用户旧版本 `is_active=0`，再 INSERT 新版本 `is_active=1`。

### jobs

```sql
CREATE TABLE jobs (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(128) NOT NULL,
  company VARCHAR(128) NOT NULL,
  city VARCHAR(64),
  salary_min INT,              -- K/月
  salary_max INT,
  degree_required VARCHAR(32),  -- 大专/本科/硕士/博士/不限
  experience_required VARCHAR(32), -- 应届/1-3年/3-5年/5+年/不限
  skills_required JSON,
  description MEDIUMTEXT,
  industry VARCHAR(64),
  sort_weight INT NOT NULL DEFAULT 0, -- 运营干预权重
  is_online TINYINT UNSIGNED NOT NULL DEFAULT 1, -- 上下架
  is_deleted TINYINT UNSIGNED NOT NULL DEFAULT 0, -- 软删
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX (is_online, is_deleted, city)
);
```

> **职责区分**：`is_online` 临时下架（招满后可恢复），`is_deleted` 永久废弃。列表默认只展示未删除。

### matches

```sql
CREATE TABLE matches (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  match_batch_id VARCHAR(64) NOT NULL,  -- 批次号，时间戳+user_id 或雪花ID
  user_id BIGINT UNSIGNED NOT NULL,
  resume_id BIGINT UNSIGNED NOT NULL,
  job_id BIGINT UNSIGNED NOT NULL,
  score TINYINT UNSIGNED NOT NULL,      -- 0-100
  reason TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX (user_id, resume_id, match_batch_id),
  INDEX (user_id, created_at)
);
```

> **历史表只增不删**：用户资料变更不清 matches 表，仅清 Redis 缓存。

### admins

```sql
CREATE TABLE admins (
  openid VARCHAR(64) PRIMARY KEY,
  role VARCHAR(32) NOT NULL DEFAULT 'normal', -- normal / super
  is_enabled TINYINT UNSIGNED NOT NULL DEFAULT 1,
  remark VARCHAR(64),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### prompts

```sql
CREATE TABLE prompts (
  `key` VARCHAR(64) PRIMARY KEY,  -- resume_gen / job_match
  template MEDIUMTEXT NOT NULL,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### admin_operation_logs

```sql
CREATE TABLE admin_operation_logs (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  admin_openid VARCHAR(64) NOT NULL,
  action VARCHAR(64) NOT NULL,       -- create_job / update_job / delete_job / ...
  target_type VARCHAR(32),           -- job / prompt
  target_id VARCHAR(64),
  detail JSON,                       -- 改动前后值
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX (admin_openid, created_at)
);
```

### 种子数据

阶段 1 建表后立即 SQL 灌入：
- 20+ 岗位，覆盖各城市/学历/经验/技能
- 默认 prompt 模板（resume_gen / job_match）

### 匹配流程

```
用户点击「匹配岗位」
        │
        ▼
  ┌─────────────────────┐
  │  1. SQL 粗筛         │
  │  必传：city           │
  │  过滤：salary 区间    │
  │  过滤：degree 层级    │
  │  过滤：experience    │
  │  模糊：position 关键词│
  │  排序：技能命中 + 权重 │
  │  → Top 15 候选       │
  └──────────┬──────────┘
             ▼
  ┌─────────────────────┐
  │  2. Redis 查列表缓存  │  命中 → 直接返
  │  key: list:{uid}:{rid}│
  └──────────┬──────────┘
             ▼
  ┌─────────────────────┐
  │  3. 逐个查单岗位缓存  │  命中的从 LLM 集合中剔除
  │  key: {uid}:{jid}    │
  └──────────┬──────────┘
             ▼
  ┌─────────────────────┐
  │  4. LLM 精排         │  未命中部分批量送 LLM
  │  max_tokens=1000     │  强制 JSON 输出
  │  超时 10s + 重试 2 次 │
  └──────────┬──────────┘
             ▼
  ┌─────────────────────┐
  │  5. 写回             │
  │  Redis 列表+单岗位    │  MySQL matches 持久化（带 batch_id）
  └──────────┬──────────┘
             ▼
  返回 Top 10 列表（按 score desc）
```

### 缓存失效

| 触发 | 动作 | 工具 |
|------|------|------|
| 用户更新资料 / 重新生成简历 | DEL `match:list:{uid}:*` + DEL `match:{uid}:*` | SCAN |
| 岗位修改 / 上下架 / 软删 | DEL `match:list:*` 中含该 jid 的 + DEL `match:*:{jid}` | SCAN |
| Prompt `job_match` 改动 | DEL 所有 `match:list:*` + 所有 `match:*:*` | SCAN |
| Prompt `resume_gen` 改动 | 不清匹配缓存 | — |

> 用 SCAN 替代 KEYS，避免阻塞主线程。

## §3 页面 UX

### 用户端流程

```
[启动小程序] → 首页（静默登录）
     ├→ [填写资料生成简历] → 5 步表单 → 简历预览 → [找岗位]
     └→ [查看岗位推荐] → 若无简历：弹确认 → 岗位列表 → 岗位详情
```

### 首页（动态态）

- **无简历**：「填写资料生成简历」「查看岗位推荐」
- **有简历**：「更新我的简历」「查看匹配岗位」+ 底部状态提示

### 资料表单（5 步）

- Step 1：最高学历
- Step 2：工作经历（可多条）
- Step 3：技能点（标签预置 + 自定义）
- Step 4：期望（城市 / 薪资 / 岗位）
- Step 5：预览提交

**降级方案**：MVP 先做 3 步核心（教育 + 经历 + 期望信息），技能用文本输入。

**草稿**：每步切换自动 `wx.setStorageSync('resume_form_draft')`，进入时优先回填。

### 简历预览

- Markdown 渲染（towxml）
- 底部：「换一版表述」「修改资料」「找岗位」

### 岗位列表

- 卡片：公司名 + 岗位 + 城市 + 薪资 + 匹配分圆环 + 文字 + 理由
- Loading 分阶段文案：粗筛 → LLM → 15s 追加提示
- 35s 前端超时
- 缓存命中时秒返
- 空状态：「当前条件下无匹配岗位」+ 「去修改期望」

### 岗位详情

- 匹配分大字 + 理由
- 岗位描述（Markdown）
- 复制按钮

### 管理端

- 入口：「我的」→「管理后台」按钮
- 前端 `onLoad` 调 `GET /api/admin/check` 拦截
- 后端 `/api/admin/*` 统一中间件鉴权
- 页面：job-list / job-edit / prompt-edit

## §4 成本 / 限流 / 部署

### LLM 成本

- 100 日活，人均 1 次操作 → 日成本 ~¥0.7，月 ~¥20
- 缓存命中进一步降低真实调用量
- 输入硬限 ≤15 岗位 + `max_tokens` 兜底

### 限流（Redis）

| 维度 | 规则 | Key |
|------|------|-----|
| 简历生成 | 每用户每天 ≤ 3 次 | `rl:resume:{uid}:{date}` |
| 匹配 | 每用户每天 ≤ 5 次 | `rl:match:{uid}:{date}` |
| 匹配防重 | 30s 内 ≤ 1 次 | `match:lock:{uid}` (NX EX 30) |
| 管理端写 | 每 admin 每分钟 ≤ 30 次 | `rl:admin:{openid}:{min}` |
| 全局 LLM | 每分钟 ≤ 60 次 | `rl:llm:global:{min}` |

**时区**：按北京自然日。

**429 文案**：区分「今日次数用完」/「操作太频繁」。

### 部署

```
腾讯云轻量 2C4G 80GB SSD  ~¥70/月
├── /opt/jianli-app/backend/   Node + Express + PM2
├── MySQL 8 同机
├── Redis 6 同机
└── Nginx 443 → 127.0.0.1:3000
    └── Let's Encrypt SSL

域名 .cn         ~¥30/年
COS 备份         <¥5/月
```

**资源上限**：

| 服务 | 配置 |
|------|------|
| MySQL | `innodb_buffer_pool_size = 512M` |
| Redis | `maxmemory 256mb` + `maxmemory-policy allkeys-lru` |
| Node | PM2 `max_memory_restart: 500M` |

**PM2**：`pm2 save` + `pm2 startup`，日志按天切割。

**certbot**：`certbot renew` 定时任务，证书自动续期。

### 备案

- 阶段 0 启动 ICP 备案（7-20 天）
- 开发期：开发者工具不校验合法域名 + IP 直连
- 上线前：备案完成 → 域名解析 + Nginx HTTPS + 小程序后台白名单

### 安全基线

- `.env` 入 `.gitignore`
- MySQL 业务账号仅当前库读写
- Redis 密码 + RDB
- SQL 参数化
- 用户输入转义防 XSS

## §5 分阶段交付

| 阶段 | 名称 | 工作日 | 必做核心 | 验收关键 |
|------|------|--------|----------|----------|
| 0 | 基建 | 3 | 服务器/账号/备案启动 | 环境命令全通 + 备案受理号 |

## §6 异常 case 清单（验收必过）

| 阶段 | 异常 | 期望 |
|------|------|------|
| 1 | 无效 token 访问受保护接口 | 401 |
| 1 | DeepSeek API 不可达 | 测试接口明确报错，不 500 |
| 3 | LLM 超时/失败 | 前端友好提示，不白屏 |
| 3 | 用户连点 4 次生成 | 第 4 次 429 |
| 5 | 粗筛 0 结果 | 友好空状态，不报错 |
| 5 | 用户无简历点匹配 | 弹确认框「是否前往填写」 |
| 5 | LLM 全失败 | 技能命中数兜底，不空页 |
| 5 | 匹配缓存命中 | 秒级返回 |
| 6 | 备份真恢复 | 数据校验一致 |
| 6 | 证书过期 | 自动续期（脚本验证） |
