# 需求文档 — 简历推荐小程序

> 项目代号：jianli-app
> 版本：V1 / MVP
> 最后更新：2026-06-27

## 1. 产品定位

面向求职者的微信小程序。用户提供基础资料，系统自动生成简历，并基于简历语义匹配人工维护的岗位库，向用户推荐合适的公司+岗位。

## 2. 目标用户

- 学历：大专 ~ 硕士
- 工作年限：应届 ~ 5 年
- 使用场景：在微信内随手打开，不愿下载独立 App

## 3. MVP 功能范围

### 3.1 必须做

| 模块 | 功能 | 用户价值 |
|------|------|----------|
| 登录 | 微信静默授权，code 换 openid | 无感进入 |
| 资料表单 | 教育/经历/技能/期望 5 步填写 | 一站式收集 |
| 简历生成 | LLM 把表单数据润色成 Markdown 简历 | 自动产出可用简历 |
| 简历预览 | Markdown 渲染、「换一版」「修改资料」 | 可调整可重生成 |
| 岗位匹配 | SQL 粗筛 + LLM 精排，输出 Top 10 | 直接拿到合适岗位 |
| 岗位详情 | 公司/岗位/薪资/匹配分/理由/复制 | 决策依据 |
| 岗位管理（管理员） | CRUD、上下架、软删 | 维护岗位库 |
| Prompt 管理（管理员） | 读 / 写两个 prompt | 调优生成质量 |

### 3.2 不做（V2 再考虑）

- 用户账号体系 / 手机号绑定
- 简历 PDF 导出
- 一键投递 / 站内沟通
- 消息推送（订阅消息 / 模板消息）
- 多语言
- 简历版本对比
- 匹配历史可视化

## 4. 用户故事

| ID | 角色 | 故事 |
|----|------|------|
| U1 | 求职者 | 我想 5 分钟内填完资料、拿到一份能投的简历 |
| U2 | 求职者 | 我想知道哪些岗位真的适合我，不想要海量不相关结果 |
| U3 | 求职者 | 我想看到岗位详情里的匹配理由，告诉我哪里合适 |
| U4 | 求职者 | 我想快速复制岗位信息去 Boss / 拉勾搜索投递 |
| A1 | 运营 | 我想新增 / 编辑 / 下架岗位，不需要懂技术 |
| A2 | 运营 | 我想调整 prompt 模板，不用动代码 |
| A3 | 管理员 | 我想知道谁在什么时候改了什么 |

## 5. 验收标准（高层）

- 用户从打开小程序到看到推荐岗位，全程 ≤ 2 分钟（不含 LLM 等待）
- 匹配结果相关性：≥ 80% 用户认为前 3 条「值得点开看」
- LLM 调用成本：日活 100 时，日成本 < ¥1
- 服务可用性：99%（MVP 阶段允许偶尔手工重启）

## 6. 非功能性需求

| 维度 | 指标 |
|------|------|
| 响应时间 | API p95 < 500ms（不含 LLM 调用） |
| 并发 | 100 同在线无压力 |
| 数据备份 | 每日全量备份，保留 7 天 |
| 安全 | HTTPS 强制、SQL 注入防护、限流防刷 |
| 兼容性 | 微信版本 ≥ 8.0 |

## 7. 风险与约束

| 风险 | 影响 | 应对 |
|------|------|------|
| ICP 备案 7-20 天 | 推迟上线 | 阶段 0 启动备案，与开发并行 |
| DeepSeek 涨价或服务异常 | LLM 成本/可用性 | prompt 配置化，可切换 provider |
| LLM 输出不稳定 | 简历/匹配质量差 | JSON 强约束 + 重试 + 兜底 |
| 微信审核驳回 | 推迟上线 | 预留 1-3 天缓冲 |
| 单人开发 | 进度延期 | 每阶段核心项先做，优化项延后 |
